# AI Context — Habit Tracker

This folder contains structured context files for any AI IDE or assistant to understand this project quickly and continue development.

## Files

| File | Purpose |
|------|---------|
| `PROJECT.md` | Project overview, tech stack, architecture, and design decisions |
| `STRUCTURE.md` | Complete file tree with descriptions of every file |
| `API.md` | All backend API endpoints with request/response shapes |
| `SCHEMA.md` | Database schema (11 tables), relationships, and conventions |
| `STYLE.md` | UI design system — colors, fonts, component patterns |
| `TODO.md` | What's done, what's next, and what's deliberately skipped |
| `SETUP.md` | How to run the project locally (Docker, frontend dev server) |

## Quick Start for AI Assistants

1. Read `PROJECT.md` for the big picture
2. Read `TODO.md` to know what to work on next
3. Read `STRUCTURE.md` to find any file quickly
4. Refer to `API.md`, `SCHEMA.md`, or `STYLE.md` as needed for specific tasks
5. The full architecture spec lives in `Architecture.md` at the project root
